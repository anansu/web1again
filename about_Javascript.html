Javascript란 무엇인가?

사용자와 상호작용하는 웹페이지를 만들고 싶다는 욕망. HTML은 정적임. 수업의 목적은 주간/야간 모드 버튼을 추가하는 것.

HTML-Javascript의 만남 1 - Script 태그

자바스크립트 실습 페이지 위의 줄은 자바스크립트를 통해 hello world와 1+1을 그려낸 것임. 핵심은 script 태그를 통해 자바스크립트를 HTML내에서 실행한다는 것임. 요런 on~들을 전부 "event"들이라고 한다. 이걸 써서 사용자와 상호작용 하는 것임.

HTML-Javascript의 만남 2 - event

이렇게 이벤트가 일어났을 때 자바스크립트를 실행시키는 것이 onclick임. 중요한 이벤트로는 "클릭" 혹은 "내용이 변했을 때(on~)"

HTML-Javascript의 만남 3 - console

파일을 만들지 않더라도 아주 가볍게 코드를 실행해야하는 상황들이 있음. 그 경우 개발자도구의 console을 활용시 편하다. 대표적으로 텍스트 세는 .length 등이 있겠지. 혹은 엄청 많은 사람들 중에 추첨해야하는 경우! 미리 작성해둔 코드들을 콘솔에서 활용할 수도 있다는 것. 미리미리 쓸만한 알고리즘들을 작성해두면 편할 것이다.

Javascript의 데이터타입 - 문자열과 숫자

자료형은 총 6개 + object라는 객체가 있음. 문자/숫자/Null/Boolean 등이 있지만 나머지는 일단 신경끄자. 숫자야 어차피 사칙연산임. 문자열을 처리할 때 사용하는 JS의 properties와 methods를 링크 에서 확인할 수 있음. 뭐 trim이든 indexof든 touppercase든 웬만한건 다 된다.

변수와 대입 연산자

대입연산자 '='. 바뀌지 않는 constant에는 대입연산자 넣어봐야 의미없다. 에러만 나올 뿐. 변수는 왜 쓰는가? 이유는 너무 많다. JS에서 변수를 선언할 때, var 변수명 = 대입하는 값 이걸로 쓰는 습관을 당분간 들이도록

웹 브라우저 제어

JS를 통해 웹브라우저를 제어해보자. div태그와 span태그를 잘 쓰면 css를 적용할 수 있게 된다. style태그는 script 태그 처럼 "이 안에 있는 것들이 CSS/JS야"라는 의미를 전달하는 HTML태그임. 선택자에 있어서 .은 클래스, #은 id. id는 식별. class는 그루핑하는 것. 제어할 태그를 선택해보자면 다음과 같다. 조건문은 쉬우니까 패스한다.
